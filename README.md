# Чат-сервер

Многопользовательский чат с системой ролей, аутентификацией и различными командами в чате.

## Требования

- **Java 25+**
- **PostgreSQL** (запущенный на localhost:5432 или настроенный в `.env`)
- **Maven** (для сборки зависимостей)

## Настройка

### 1. Создайте базу данных

```sql
CREATE DATABASE "otus-java-basic-2025";
```

### 2. Настройте `.env` файл

Создайте файл `.env` в корне проекта:

```env
DB_HOST=localhost
DB_PORT=5432
DB_NAME=otus-java-basic-2025
DB_USER=postgres
DB_PASSWORD=
```

### 3. Запустите сервер

При первом запуске сервер **автоматически создаст** таблицы `roles` и `users` и добавит тестового пользователя `admin`.

---

## Как подключиться к чату

1. **Запустите сервер** (ServerApp)
2. **Запустите клиент** (ClientApp)
3. **Войдите или зарегистрируйтесь:**
    - Для входа: `/auth <nickname> <password>`
    - Для регистрации: `/reg <nickname> <password> <username>`

### Тестовый аккаунт

При первом запуске автоматически создаётся владелец:

| Nickname | Пароль | Имя в чате    | Роль  |
|----------|--------|---------------|-------|
| admin    | admin  | Administrator | OWNER |

**Пример входа:**

```
/auth admin admin
```

**Пример регистрации нового пользователя:**

```
/reg bob 123 Bob
```

### Требования к регистрации

- **Nickname:** 3-50 символов, только буквы, цифры, `_`, `-`
- **Пароль:** минимум 3 символа
- **Username:** не может быть пустым
- Nickname и username должны быть уникальными

---

## Роли пользователей

В чате существует три роли с разными уровнями доступа:

### OWNER (Владелец)

- **Префикс:** `~`
- **Как получить:** Роль назначается в базе данных (по умолчанию `admin`)
- **Права:** Все команды доступны

### ADMIN (Администратор)

- **Префикс:** `@`
- **Как получить:** Владелец выдаёт права командой `/op`
- **Права:** Большинство команд, кроме `/deop`

### USER (Пользователь)

- **Префикс:** (нет)
- **Как получить:** Автоматически при регистрации
- **Права:** Базовые команды чата

---

## Команды чата

### Команды для всех пользователей

| Команда                             | Описание                                | Пример            |
|-------------------------------------|-----------------------------------------|-------------------|
| `/help`                             | Показать список всех команд             | `/help`           |
| `/info`                             | Показать информацию о себе (имя, роль)  | `/info`           |
| `/name <новое_имя_пользователя>`    | Сменить своё имя пользователя           | `/name SuperUser` |
| `/w <имя_пользователя> <сообщение>` | Отправить личное сообщение пользователю | `/w Bob Привет!`  |
| `/exit`                             | Выйти из чата                           | `/exit`           |

### Команды для ADMIN и OWNER

| Команда                    | Доступ       | Описание                             | Пример                           |
|----------------------------|--------------|--------------------------------------|----------------------------------|
| `/all <сообщение>`         | ADMIN, OWNER | Отправить сообщение от имени сервера | `/all Сервер будет перезагружен` |
| `/kick <имя_пользователя>` | ADMIN, OWNER | Отключить пользователя из чата       | `/kick Bob`                      |
| `/op <имя_пользователя>`   | ADMIN, OWNER | Выдать права администратора          | `/op Bob`                        |

### Команды только для OWNER

| Команда                    | Доступ | Описание                   | Пример      |
|----------------------------|--------|----------------------------|-------------|
| `/deop <имя_пользователя>` | OWNER  | Снять права администратора | `/deop Bob` |

---

## Ограничения команд

- **Нельзя выгнать владельца** (`/kick`)
- **Нельзя снять права у владельца** (`/deop`)
- **Нельзя выгнать самого себя**
- **Нельзя отправить личное сообщение самому себе**
- **Имя_пользователя не может:**
    - Быть пустым
    - Содержать пробелы
    - Начинаться с `/`
    - Совпадать с именем другого пользователя

---

## Примеры использования

### Базовый сценарий

1. **Первый пользователь (OWNER из БД):**
   ```
   /auth admin admin
   ~Administrator выполнил вход
   > ~Administrator присоединился к чату
   ```

2. **Второй пользователь (регистрация):**
   ```
   /reg bob 123 Bob
   ~Bob выполнил вход
   > ~Bob присоединился к чату
   ```

3. **Владелец выдаёт права Bob:**
   ```
   /op Bob
   > @Bob теперь администратор
   ```

4. **Общение:**
   ```
   ~Administrator: Привет всем!
   @Bob: привет
   ~Tom: Привет!
   ```

### Смена имени пользователя

```
/name SuperAdmin
* Имя пользователя изменено: SuperAdmin
> ~Administrator теперь известен как ~SuperAdmin
```

При следующем входе `/auth admin admin` пользователь войдёт с новым именем `SuperAdmin`.

### Личные сообщения

```
/w Bob Это личное сообщение
* Сообщение отправлено пользователю Bob
```

Bob получит:

```
> ЛС от Administrator: Это личное сообщение
```

### Сообщение от сервера

```
/all Внимание! Важное объявление
[СЕРВЕР] Внимание! Важное объявление
```

## Визуальные префиксы

В чате пользователи отображаются с префиксами ролей:

- `~Administrator: сообщение` — владелец
- `@Bob: сообщение` — администратор
- `Tom: сообщение` — обычный пользователь

Префиксы отображаются везде:

- При входе: `~Administrator выполнил вход`
- При присоединении: `> ~Administrator присоединился к чату`
- При отключении: `> ~Administrator покинул чат`
- В сообщениях: `~Administrator: привет`

## Технические детали

### Безопасность

- **BCrypt хэширование:** Пароли хранятся в БД в виде хэшей
- **Защита от SQL injection:** Используются PreparedStatement
- **Валидация данных:** Проверка nickname, password, username при регистрации

### Персистентность

Все данные хранятся в PostgreSQL:

- **Роли:** Таблица `roles` (OWNER, ADMIN, USER)
- **Пользователи:** Таблица `users` (nickname, username, password_hash, role_id)
- **Изменения сохраняются:** Смена username (`/name`) и роли (`/op`, `/deop`) обновляют БД

### Автоматическая инициализация

При первом запуске сервер:

1. Создаёт таблицы `roles` и `users` (если не существуют)
2. Вставляет роли (OWNER, ADMIN, USER)
3. Создаёт пользователя `admin/admin` с ролью OWNER

Можно безопасно удалить таблицы и перезапустить сервер - всё пересоздастся автоматически.

## Заметки

- При отключении владельца права **НЕ передаются** автоматически
- Роли хранятся в БД и сохраняются между перезапусками
- Nickname используется для входа и **не меняется**
- Username отображается в чате и **может меняться** через `/name`
- Все изменения (роли, username) сохраняются в БД
